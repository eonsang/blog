# MDX로 인터랙티브 블로그 만들기

MDX는 마크다운 안에서 JSX를 사용할 수 있게 해주는 포맷입니다.

## MDX란?

일반 마크다운의 확장으로, React 컴포넌트를 문서 안에 삽입할 수 있습니다.

```mdx
# 제목

일반 마크다운 문법을 그대로 사용하면서

<CustomChart data={chartData} />

컴포넌트도 함께 렌더링할 수 있습니다.
```

## Velite와 함께 사용하기

Velite는 콘텐츠를 타입 안전하게 관리할 수 있는 도구입니다.

```javascript
// velite.config.js
export default defineConfig({
  collections: {
    posts: {
      schema: s.object({
        title: s.string(),
        content: s.mdx(),
      }),
    },
  },
});
```

## rehype 플러그인

코드 구문 강조, 자동 링크 등 다양한 플러그인을 조합할 수 있습니다:

- **rehype-pretty-code**: 코드 블록 구문 강조
- **rehype-slug**: 헤딩에 ID 자동 추가
- **rehype-autolink-headings**: 헤딩에 앵커 링크 추가

## 장점

1. **친숙한 문법**: 마크다운을 이미 알고 있다면 바로 시작 가능
2. **컴포넌트 재사용**: React 생태계의 모든 컴포넌트 활용 가능
3. **타입 안전성**: Velite와 함께 사용하면 빌드 타임에 검증
4. **SEO 친화적**: 정적 생성으로 최적의 성능

기술 블로그를 운영한다면 MDX는 최고의 선택입니다.
