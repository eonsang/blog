노트북 없이 외출했는데 급하게 코드를 수정해야 할 때가 있다. 혹은 이동 중에 떠오른 아이디어를 바로 구현하고 싶을 때도 있다. Claude Code를 활용하면 스마트폰만으로도 꽤 수준 높은 개발 작업이 가능하다.

## 왜 모바일에서 Claude Code인가

모바일에서 코딩한다고 하면 대부분 회의적이다. 작은 화면, 불편한 키보드, 제한된 멀티태스킹. 하지만 Claude Code는 자연어로 지시하고 AI가 코드를 작성하는 구조이기 때문에, 직접 타이핑하는 양이 적다. 핵심은 **무엇을 만들지 설명하는 것**이지 코드를 한 줄씩 치는 게 아니다.

## 접속 방법

### 1. SSH 클라이언트 활용

가장 안정적인 방법이다. 개발 서버나 클라우드 인스턴스에 SSH로 접속하면 된다.

**iOS 추천 앱:**
- Termius
- Blink Shell

**Android 추천 앱:**
- Termux
- JuiceSSH

```bash
# 서버에 Claude Code가 설치된 상태에서
ssh user@your-server.com
claude
```

접속 후 Claude Code CLI를 실행하면 데스크톱과 동일한 환경에서 작업할 수 있다.

### 2. GitHub Codespaces

브라우저 기반이라 별도 앱 설치 없이도 사용 가능하다. GitHub 모바일 앱에서 Codespace를 열거나, 모바일 브라우저에서 직접 접속한다.

```bash
# Codespace 터미널에서
npm install -g @anthropic-ai/claude-code
claude
```

### 3. VS Code Server (code-server)

자체 서버에 code-server를 설치하면 모바일 브라우저에서 VS Code 환경을 그대로 사용할 수 있다. 터미널 패널에서 Claude Code를 실행하면 된다.

## 모바일에서 효과적으로 작업하는 팁

### 프롬프트를 구체적으로 작성하라

데스크톱에서는 결과를 보고 즉시 수정 지시를 내릴 수 있지만, 모바일에서는 매번 타이핑이 번거롭다. 처음부터 구체적으로 지시하면 반복 작업을 줄일 수 있다.

```
// 모호한 지시
"로그인 기능 만들어줘"

// 구체적인 지시
"src/app/auth에 이메일/비밀번호 로그인 API 라우트를 만들어줘.
bcrypt로 비밀번호 검증하고, JWT 토큰을 httpOnly 쿠키로 설정해.
실패 시 적절한 에러 응답을 반환하도록 해."
```

한 번의 프롬프트로 원하는 결과에 가까워질수록 모바일 환경의 불편함이 줄어든다.

### CLAUDE.md를 미리 세팅해두라

프로젝트 루트에 `CLAUDE.md` 파일을 잘 작성해두면, 매번 프로젝트 컨텍스트를 설명할 필요가 없다. 모바일에서 긴 설명을 타이핑하는 수고를 아낄 수 있다.

```markdown
# CLAUDE.md 예시

## 프로젝트 개요
Next.js 15 + TypeScript 블로그 프로젝트

## 기술 스택
- Framework: Next.js (App Router)
- Styling: Tailwind CSS
- Database: PostgreSQL + Prisma
- Auth: NextAuth.js

## 컨벤션
- 컴포넌트: PascalCase
- 유틸 함수: camelCase
- API 응답: snake_case
- 커밋 메시지: conventional commits (한국어)

## 테스트
- npm run test로 vitest 실행
- npm run lint로 ESLint 실행
```

### 텍스트 대치(단축어)를 활용하라

자주 쓰는 프롬프트 패턴을 스마트폰의 텍스트 대치 기능에 등록해두면 편하다.

| 단축어 | 변환 결과 |
|--------|-----------|
| `ccfix` | `이 에러를 분석하고 수정해줘:` |
| `cctest` | `이 파일에 대한 테스트 코드를 작성해줘` |
| `ccrefac` | `이 코드를 리팩토링해줘. 기능은 동일하게 유지해` |
| `ccreview` | `이 변경사항을 리뷰해줘. 버그나 개선점을 알려줘` |

iOS의 텍스트 대치, Android의 개인 사전 기능을 활용하면 된다.

### 슬래시 커맨드를 적극 활용하라

긴 문장을 타이핑하는 대신 Claude Code의 슬래시 커맨드를 사용하면 효율적이다.

```bash
# 현재 상태 확인
/status

# 비용 확인
/cost

# 컨텍스트 초기화
/clear

# 커밋
/commit
```

### 음성 입력을 활용하라

모바일 키보드의 음성 입력 기능을 활용하면 긴 프롬프트도 빠르게 입력할 수 있다. Claude Code는 자연어를 이해하기 때문에 말하듯이 지시해도 잘 동작한다.

## 모바일에 적합한 작업 유형

모바일에서 모든 개발 작업이 동일하게 효율적이지는 않다. 아래는 모바일 환경에서 특히 잘 맞는 작업들이다.

**잘 맞는 작업:**
- 버그 수정 — 에러 메시지를 붙여넣고 수정 요청
- 단일 파일 수정 — 특정 함수 리팩토링, 타입 수정
- 코드 리뷰 — PR diff를 분석하고 피드백 받기
- 커밋 & PR 생성 — `/commit` 명령으로 간편하게
- 문서 작성 — README, API 문서 등
- 설정 변경 — 환경 변수, config 파일 수정

**데스크톱이 나은 작업:**
- 대규모 리팩토링 — 여러 파일을 동시에 확인해야 할 때
- UI 작업 — 시각적 결과를 확인하며 반복해야 할 때
- 디버깅 — 복잡한 로그를 추적해야 할 때

## 실전 예시: 모바일에서 블로그 포스트 추가

실제로 이 글도 모바일 환경을 상정하고 작성 가능한 수준의 작업이다.

```bash
claude

> content/posts 디렉토리에 새 MDX 블로그 포스트를 만들어줘.
> 제목은 "모바일에서 Claude Code로 개발하기"이고,
> 모바일 환경에서 SSH나 Codespaces로 접속해서
> Claude Code를 활용하는 방법을 설명하는 글이야.
```

이 한 번의 프롬프트로 파일 생성부터 frontmatter 설정, 콘텐츠 작성까지 처리된다.

## 네트워크 끊김 대비

모바일 환경에서는 네트워크가 불안정할 수 있다. 몇 가지 대비책을 마련해두자.

- **tmux 또는 screen 사용**: SSH 접속이 끊겨도 세션이 유지된다
- **자주 커밋하기**: 작업 단위를 작게 나누고 자주 커밋하면 손실을 최소화할 수 있다
- **Mosh 사용**: SSH 대신 Mosh를 사용하면 네트워크 전환(Wi-Fi ↔ LTE)에도 세션이 유지된다

```bash
# tmux 세션 생성
tmux new -s dev

# 재접속 시 세션 복구
tmux attach -t dev
```

## 마무리

모바일에서의 개발은 데스크톱을 대체하는 것이 아니다. **보완하는 것**이다. 급한 핫픽스, 이동 중 아이디어 구현, 가벼운 코드 리뷰 같은 상황에서 Claude Code와 모바일 환경의 조합은 충분히 실용적이다.

핵심은 세 가지다:
1. `CLAUDE.md`로 프로젝트 컨텍스트를 미리 세팅해두기
2. 구체적인 프롬프트로 반복 작업 줄이기
3. tmux + Mosh로 안정적인 세션 유지하기

노트북을 꺼낼 수 없는 상황에서도 개발을 멈추지 않아도 된다.
